Found something broken? Thank you for telling us.

## Where to report

**[Submit a bug report →](https://majksa.notion.site/33b93506081880de83b1ed40e3759e46)**

That form routes your report directly into our triage queue. Please use it in preference to opening GitHub issues or pinging us on Discord — it captures the information we need and we won't lose track of it.

## How to write a good bug report

A good bug report answers three questions: **what did you do**, **what happened**, and **what did you expect**?

- **Be specific.** "The RSVP button didn't work" is much less useful than "On event ID `evt_abc123`, clicking **Yes** showed a red toast that said 'Network error' at about 19:40 UTC on 2026-04-17".
- **Include one-line repro steps.** Number them. If we can reproduce, we can fix.
- **Attach screenshots** if the issue is visual. The form supports file uploads.
- **Say what browser / OS / device you're on** if it might matter.
- **Paste any error message verbatim.** Don't paraphrase error codes or tags.
- **One bug per report.** If you found three, file three reports. They often have different fixes.

## What happens next

1. We triage new reports within a few days.
2. You get a link to the tracked task; updates flow there.
3. If we need more info, we'll comment on the task.
4. When it's fixed, the task is marked resolved and the fix appears in the next [changelog](/changelog/) entry.
