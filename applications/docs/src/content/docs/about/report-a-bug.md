Found something broken? Thank you for telling us.

## Where to report

The quickest path is the **Report a bug** link in the app: open the user menu in the sidebar of the web app and click **Report a bug**. It opens the GitHub issue form in a new tab.

Alternatively, go straight to **[GitHub Issues →](https://github.com/maxa-ondrej/sideline/issues/new)**

## How to write a good bug report

A good bug report answers three questions: **what did you do**, **what happened**, and **what did you expect**?

- **Be specific.** "The RSVP button didn't work" is much less useful than "On event ID `evt_abc123`, clicking **Yes** showed a red toast that said 'Network error' at about 19:40 UTC on 2026-04-17".
- **Include one-line repro steps.** Number them. If we can reproduce, we can fix.
- **Attach screenshots** if the issue is visual. GitHub issues support image uploads.
- **Say what browser / OS / device you're on** if it might matter.
- **Paste any error message verbatim.** Don't paraphrase error codes or tags.
- **One bug per report.** If you found three, file three reports. They often have different fixes.

## What happens next

1. We triage new issues within a few days.
2. Updates and discussion happen on the GitHub issue itself — subscribe to it for notifications.
3. If we need more info, we'll comment on the issue.
4. When it's fixed, the issue is closed and the fix appears in the next [changelog](/changelog/) entry.
