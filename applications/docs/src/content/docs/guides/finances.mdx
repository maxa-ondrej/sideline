Sideline's Finance module lets admins and members with the Treasurer role define fees (membership dues, equipment levies, tournament entry costs), assign them to members, record payments as they come in, and track team expenditures. Members can check their own balance at any time — in Discord or in the web app.

## Key concepts

- **Fee** — a named charge defined at the team level (e.g. "Spring 2026 membership", "Kit fee"). Each fee has a default amount, a currency, and an optional due date.
- **Assignment** — the record that links a fee to a specific member. An assignment inherits the fee's amount and due date, but both can be overridden per member.
- **Payment** — money received against an assignment. Payments are never deleted; they can be **voided** with a reason if recorded in error.
- **Status** — computed automatically from the assignment's amounts and due date:

  | Status | Meaning |
  |---|---|
  | **Pending** | No payment received yet, due date has not passed |
  | **Partial** | Some payment received, not yet fully paid |
  | **Paid** | Fully paid |
  | **Overdue** | No or partial payment and the due date has passed |
  | **Waived** | Captain has marked this assignment as waived (e.g. hardship exemption) |

## Who can do what

Finance permissions follow a treasurer-style split across four built-in roles. By default:

| Action | Admin | Captain | Treasurer | Player |
|---|---|---|---|---|
| View fee list and assignments | Yes | Yes | Yes | No |
| View finance overview (all members) | Yes | Yes | Yes | No |
| View balance summary (Income / Expenses / Net) | Yes | Yes | Yes | No |
| Create, edit, or archive fees | Yes | No | Yes | No |
| Assign fees to members | Yes | No | Yes | No |
| Record or void payments | Yes | No | Yes | No |
| Create, edit, or delete expenses | Yes | No | Yes | No |
| View own fee status (My Payments) | Yes | Yes | Yes | Yes |

All team members can access their own My Payments page and check their own fee balance in Discord, regardless of role.

<Aside type="note" title="Planned features">
Several Finance features are planned for follow-up releases:

- **Monthly recurrence** — auto-generating monthly charges from a fee definition.
- **Self-report "I've paid"** — a Discord interaction for members to notify captains of unrecorded payments.
- **Expense receipts** — attaching an image or document URL to an expense record.
</Aside>

## Finance overview page

Go to **Team → Finances** (`/teams/:teamId/finances`) to see the full financial picture for your team. The page is visible to anyone holding `finance:view` (Admin, Captain, and Treasurer by default). The active tab is reflected in the URL so you can bookmark or share a specific view.

The page has three tabs:

**Overview tab** (default) — three KPI tiles showing team-wide totals per currency:
- **Income** — sum of all non-voided payments received
- **Expenses** — sum of all recorded expenditures
- **Net** — Income minus Expenses (may be negative)

Below the tiles, a summary table lists all active members with their total due, total paid, and assignment status counts. Use this view to quickly see who is behind on payments.

**By assignment tab** — a flat list of individual payment records. Use the filter controls to narrow by member, fee, date range, or to include voided payments.

**Expenses tab** — a list of all team expenditures. From this tab you can add, edit, and delete expense records (requires `finance:manage_fees`).

## Fee management page

Go to **Team → Finances → Fees** (`/teams/:teamId/finances/fees`) to create, edit, and archive fees and to assign fees to members. This page requires the `finance:manage_fees` permission (Admin or Treasurer by default).

From this page you can:
- Create a new fee (name, amount, currency, due date, scope).
- Assign an existing fee to individual members, with optional per-member amount or due-date overrides.
- Edit a fee's details or archive it when it is no longer needed.
- Waive an individual assignment (e.g. hardship exemption).
- Record a payment directly from the assignment list.

## Expense tracking

Use the **Expenses tab** on the Finances page (`/teams/:teamId/finances`) to record and manage team expenditures — pitch hire, equipment purchases, travel costs, tournament entry fees, and anything else the team spends money on.

Expense management requires the `finance:manage_fees` permission (Admin or Treasurer by default). Members with only `finance:view` (Captain) can see the balance summary but cannot create or modify expenses.

### Expense categories

Every expense must be assigned one of the following categories:

| Category | Typical use |
|---|---|
| **Fields** | Pitch or venue hire |
| **Equipment** | Kit, cones, balls, bibs, and other gear |
| **Travel** | Transport costs for away fixtures |
| **Tournaments** | Entry fees for competitions |
| **Other** | Anything that does not fit the above |

### Recording an expense

<Steps>

1. Navigate to **Team → Finances** and open the **Expenses tab**.
2. Click **Add expense**.
3. Fill in the category, amount, currency, date, and a short description (max 500 characters).
4. Click **Save**. The expense appears in the list and the balance summary updates immediately.

</Steps>

### Editing or deleting an expense

Click any expense row to open the edit form. You can update any field. To remove the record, click **Delete** inside the form.

<Aside type="caution" title="Deletions are permanent">
  Expenses are hard-deleted — they cannot be restored from the UI once removed. An audit record is always kept internally, but it is not accessible through the web app.
</Aside>

### Balance summary

The **Overview tab** on the Finances page shows three KPI tiles — **Income**, **Expenses**, and **Net** — aggregated per currency. Income is the total of all non-voided fee payments; Expenses is the total of all recorded expense amounts; Net is Income minus Expenses.

You can filter the summary to a specific date range using the `from` and `to` query parameters via the API (see the [API section below](#captainadmin-workflows-via-api)).

### Expense API (for automation)

Admins and Treasurers can also manage expenses via the REST API.

#### List expenses

```http
GET /api/teams/{teamId}/expenses
```

Optional query parameters: `category`, `from` (ISO 8601), `to` (ISO 8601).

#### Create an expense

```http
POST /api/teams/{teamId}/expenses
Content-Type: application/json

{
  "amountMinor": 15000,
  "currency": "CZK",
  "spentAt": "2026-05-10T00:00:00.000Z",
  "category": "fields",
  "description": "Pitch hire — spring season round 4"
}
```

`amountMinor` must be a positive integer (minor currency units). The request returns `201 Created` with the full `ExpenseView` on success.

#### Update an expense

```http
PATCH /api/teams/{teamId}/expenses/{expenseId}
Content-Type: application/json

{
  "description": "Pitch hire — spring season round 4 (corrected)"
}
```

All fields are optional. If you supply `currency` you must also supply `amountMinor`.

#### Delete an expense

```http
DELETE /api/teams/{teamId}/expenses/{expenseId}
```

Returns `204 No Content` on success.

#### Balance summary

```http
GET /api/teams/{teamId}/finances/balance-summary?from=2026-01-01T00:00:00.000Z&to=2026-12-31T23:59:59.000Z
```

Returns `BalanceSummary[]` — one element per currency — with `incomeMinor`, `expensesMinor`, and `netMinor` fields.

## My Payments page

Any team member can visit **Team → My Payments** (`/teams/:teamId/my-payments`) to see a personal summary of all their fee assignments. No special permission is required — every member sees only their own data.

### Four KPI cards

| Card | What it shows |
|---|---|
| **Outstanding** | Total amount still owed (one entry per currency; additional currencies shown as "+N more") |
| **Overdue** | Count of assignments past their due date with unpaid balance |
| **Paid total** | Sum of all payments made (per currency) |
| **Next due** | Amount and due date of the earliest upcoming pending or partial assignment |

### Filter chips

Below the KPI cards a row of chips lets you narrow the assignment list:

- **All** — every assignment
- **Outstanding** — pending, partial, or overdue assignments
- **Paid** — fully paid assignments
- **Waived** — assignments the captain has waived

### Per-fee table

Assignments are grouped by currency. Each group is displayed as a table with columns for **Fee name**, **Due date**, **Amount paid**, and **Status badge**. Click the chevron on any row that has recorded payments to expand an inline payment history showing the amount, method, date, and who recorded each payment.

### Outstanding / overdue alert banner

When a member has outstanding or overdue fees, a banner appears at the top of the **Team Dashboard** page with a summary of the amounts owed and a direct link to the My Payments page.

## Checking your own balance in Discord

Any member can check their own fee status using the `/finance status` slash command.

<Steps>

1. Open Discord in any channel where the Sideline bot has permission.
2. Type `/finance status` and press Enter.
3. The bot replies with an ephemeral (visible only to you) embed.

</Steps>

The embed colour tells you at a glance where you stand:

- **Green** — all fees paid or waived.
- **Amber** — one or more fees are pending or partially paid.
- **Red** — one or more fees are overdue.

Each outstanding fee appears as a field showing the fee name, amount still owed, and due date. Overdue items also show how many days past due they are.

<Aside type="caution" title="No access? Check with your captain">
  If `/finance status` tells you "not a member", your captain needs to make sure your Sideline profile is linked to your Discord account. If you have no fees assigned at all, the bot shows the green all-clear embed.
</Aside>

## Payment reminders

Sideline automatically sends you a Discord DM when a fee assignment reaches one of five reminder points relative to its due date:

| When | Reminder |
|---|---|
| 3 days before due | "Heads up — payment due soon" (blue embed) |
| On the due date | "Payment due today" (yellow embed) |
| 3 days overdue | "Payment overdue" (red embed) |
| 10 days overdue | "Payment overdue" (red embed) |
| 21 days overdue | "Payment overdue" (red embed) |

Each DM shows the fee name, total amount, how much is still outstanding, and the due date. Reminders stop automatically once the assignment is marked paid or waived.

<Aside type="note" title="Reminders require Discord DMs">
Reminders are sent as Discord direct messages. If you have DMs from server members disabled in your Discord privacy settings, you will not receive them. Check **User Settings → Privacy & Safety → Allow direct messages from server members** if you are not getting reminders you expect.
</Aside>

### Payment due dates in your calendar

Your personal iCal feed (see [Calendar subscription](/guides/calendar-subscription/)) also includes an all-day event for each unpaid or overdue fee assignment, with a built-in alarm that fires one day before the due date. This gives you a second reminder channel alongside the Discord DMs, and lets you see upcoming payment deadlines alongside team events.

## Captain / admin fee workflows (via API)

Most fee management tasks are available through the web UI (see the [Fee management page](#fee-management-page) section above). For scripted or bulk operations, admins and treasurers can also use the REST API directly. See the [API overview](/api/overview/) for authentication instructions. For expense API examples see the [Expense API](#expense-api-for-automation) section above.

### Create a fee

```http
POST /api/teams/{teamId}/fees
Content-Type: application/json

{
  "name": "Spring 2026 membership",
  "amountMinor": 50000,
  "currency": "CZK",
  "dueAt": "2026-03-31T23:59:59.000Z",
  "targetScope": "all_members",
  "description": null
}
```

`amountMinor` is the amount in the smallest unit of the currency (e.g. hellers for CZK, cents for EUR). 50 000 hellers = 500 CZK.

### Assign a fee to members

```http
POST /api/teams/{teamId}/fees/{feeId}/assignments
Content-Type: application/json

{
  "memberIds": ["member-uuid-1", "member-uuid-2"],
  "amountMinorOverride": null,
  "dueAtOverride": null
}
```

Pass `amountMinorOverride` or `dueAtOverride` to override the fee's defaults for specific members. Pass `null` to use the fee-level values.

### Record a payment

```http
POST /api/teams/{teamId}/fees/{feeId}/assignments/{assignmentId}/payments
Content-Type: application/json

{
  "amountMinor": 50000,
  "method": "cash",
  "paidAt": "2026-02-15T10:30:00.000Z",
  "note": "Paid at training"
}
```

`method` is either `"cash"` or `"bank_transfer"`.

### Void a payment

```http
DELETE /api/teams/{teamId}/payments/{paymentId}
Content-Type: application/json

{
  "reason": "Entered against wrong member"
}
```

The payment is marked voided (not deleted). The assignment's paid amount is automatically recalculated.

### Waive an assignment

```http
PATCH /api/teams/{teamId}/fees/{feeId}/assignments/{assignmentId}
Content-Type: application/json

{
  "waived": true,
  "waivedReason": "Hardship exemption approved by committee"
}
```

### Archive a fee

Archiving a fee hides it from active listings but preserves all assignments and payments for historical records.

```http
DELETE /api/teams/{teamId}/fees/{feeId}
```

Archived fees cannot be modified. Create a new fee if you need to issue a revised charge.
